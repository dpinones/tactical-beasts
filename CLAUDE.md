## Project Overview

Dojo Starter React â€” a template for building on-chain apps with **Dojo Engine + Starknet + React**. It provides dual authentication (Cartridge Controller + burner/guest accounts), type-safe contract interaction via auto-generated bindings, and Torii indexer integration for GraphQL queries.

## Commands

```bash
npm install --legacy-peer-deps   # Install dependencies (--legacy-peer-deps is required)
npm run dev                      # Start Vite dev server (localhost:5173)
npm run build                    # TypeScript check + Vite production build
npm run test                     # Run tests with Vitest
npm run generate                 # Generate TS bindings from Cairo contracts via sozo
```

Requires Node >= 20. Local development also requires Katana (localhost:5050) and Torii (localhost:8080) running, unless using a Slot deployment via `VITE_SLOT_INSTANCE`.

## Architecture

### Provider Stack (src/main.tsx)

The app initializes asynchronously via `setup()` before rendering. Providers are nested in this order:

```
StarknetProvider â†’ QueryClientProvider â†’ ChakraBaseProvider â†’ WalletProvider â†’ DojoProvider â†’ BrowserRouter
```

If `setup()` fails (e.g., Katana/Torii not running), a fallback error UI is rendered instead.

### Core Dojo Layer (src/dojo/)

- **setup.ts** â€” Async initialization: creates ToriiClient, DojoProvider, BurnerManager, and wires up the world from generated bindings. Returns `SetupResult` passed to context providers.
- **DojoContext.tsx** â€” Provides `useDojo()` hook. Returns `{ setup, account, masterAccount }`. The `account` object wraps burner/controller account with `create`, `list`, `select`, `clear` methods.
- **WalletContext.tsx** â€” Manages authentication state machine. Tracks `connectionStatus` ("selecting" | "connecting_burner" | "connecting_controller") and persists `accountType` to localStorage. Guest mode (`connectAsGuest`) is only available when `VITE_CHAIN` is not "mainnet" or "sepolia".
- **accountStore.tsx** â€” Zustand store for active account state, synced by DojoContext effects.
- **controller/** â€” Cartridge Controller configuration. `policies.ts` auto-generates session key policies by introspecting the generated contract bindings. `controller.ts` configures chain ID (mainnet/sepolia/slot).

### Generated Files (src/dojo/typescript/) â€” DO NOT EDIT MANUALLY

These are regenerated by `npm run generate` from `manifest_slot.json`:

- **contracts.gen.ts** â€” Typed system call functions (execute transactions + view calls)
- **defineContractComponents.ts** â€” RECS component definitions for Torii sync
- **models.gen.ts** â€” TypeScript interfaces matching Cairo models

After regenerating, the hardcoded namespace string must be replaced with `DOJO_NAMESPACE` from env (see docs/adapting-contracts.md).

### Endpoint Resolution (src/config/cartridgeUrls.ts)

- With `VITE_SLOT_INSTANCE` set: URLs resolve to `https://api.cartridge.gg/x/{instance}/katana|torii`
- Without it: defaults to `localhost:5050` (RPC) and `localhost:8080` (Torii)

### Key Patterns

**Executing contract calls:**
```tsx
const { setup: { client }, account: { account } } = useDojo();
await client.system_name.method_name(account, ...args);
```

**With loading/error state:** Use `useContractActions()` hook from `src/hooks/useContractActions.ts`.

**GraphQL queries to Torii:** Use `graphql-request` with the `graphqlUrl` from `cartridgeUrls.ts`. Query names follow `{namespace}{ModelName}Models`.

**Adding new pages:** Create in `src/pages/`, register route in `src/AppRoutes.tsx`.

## Environment Variables

Configured via `.env` (see `.env_example`):

| Variable | Purpose |
|---|---|
| `VITE_SLOT_INSTANCE` | Cartridge Slot deployment name (blank = local) |
| `VITE_DOJO_NAMESPACE` | Namespace for contract calls |
| `VITE_MASTER_ADDRESS` / `VITE_MASTER_PRIVATE_KEY` | Master account for burner creation |
| `VITE_CHAIN` | "mainnet", "sepolia", or blank for Slot |
| `VITE_CONTROLLER_PRESET` | Cartridge Controller preset name |
| `VITE_ENABLE_VRF` | Enable VRF policy ("true"/"false") |

## Adapting to Custom Contracts

1. Build Cairo contracts with `sozo build` to get `manifest.json`
2. Copy manifest to `manifest_slot.json`
3. Run `npm run generate` to regenerate TypeScript bindings
4. Replace hardcoded namespace strings in generated files with `DOJO_NAMESPACE` env variable
5. Use the new typed client methods via `useDojo()`

See `docs/adapting-contracts.md` for detailed steps.
